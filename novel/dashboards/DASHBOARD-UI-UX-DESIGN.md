# 巡の運命診断室ダッシュボード UI/UX 設計書

> **バージョン**: 1.0
> **作成日**: 2026年2月14日
> **元資料**: `DASHBOARD.md`, `HANDOVER-2026-02-14.md`, MANIFEST.json

---

## 1. 設計方針

### 1.1 全体コンセプト
- **DASHBOARD.md を完全内包**: 120話の統計・3部構成・13章・さくら回想20回・キャラクター成長弧・テーマ構造を一元的に把握できる
- **引き継ぎの一元化**: SSoT（九条巡 1990-03-02）、話・さくらの教え・伏線、本編ドラフト管理、伏線逆算を統合
- **和風デザイン**: ベージュ・明朝体（Shippori Mincho）、静謐で執筆に集中できる雰囲気

### 1.2 設計原則
1. **SSoT の可視化**: 正典情報（九条巡生年月日等）を常に参照可能に
2. **階層的ナビゲーション**: オーバービュー → 詳細（話/キャラ/教え/伏線）→ 個別
3. **執筆支援**: Phase 1-3、本編ドラフト、伏線逆算で「今書くべきこと」が一目で分かる
4. **120話スケール**: 全体像を見失わない統計・章・部のサマリー

## 9. キャラクター詳細・レビュー機能

### 9.1 レビュー状況（ダブルチェック）
- **人間によるレビュー**: チェックボックス（localStorage に保存）
- **ツールによるレビュー**: チェックボックス（`npx tsx tools/verify-ep1-10.ts` / `verify-storyline.ts` 実行後チェック）

### 9.2 ペルソナ・キャラ設計変更レビュー
ストーリーライン変更後に以下を確認:
- ペルソナ（人物像）の見直しが必要
- キャラクター設計（年表・核・他キャラ関係）の見直しが必要
- メモ欄（変更点・懸念事項）
