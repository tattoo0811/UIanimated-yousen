# Roadmap: 巡の運命診断室ダッシュボード

## Overview

ダッシュボードを「正典（DASHBOARD.md）の内容をWeb UIで閲覧可能にする」というゴールへ向けて、全体統計から始めて詳細な可視化機能へと段階的に構築していく。各フェーズは正典の特定セクションに対応し、ユーザーは物語の統計、構造、テーマを多角的に探索できるようになる。

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3, 4): v1.0 マイルストーンの計画済み作業

- [ ] **Phase 1: 全体統計拡張** - ダッシュボードの基礎となる全体統計を表示
- [ ] **Phase 2: 13章構造可視化** - 物語の中核構造をタイムラインで表示
- [ ] **Phase 3: さくら回想シーン** - さくらの回想シーン20回を詳細可視化
- [ ] **Phase 4: サブテーマ統計** - テーマ別統計と3部間の推移を表示

## Phase Details

### Phase 1: 全体統計拡張
**Goal**: 120話全体の統計情報と基本構造をダッシュボードで閲覧可能にする
**Depends on**: なし（最初のフェーズ）
**Requirements**: OV-01, OV-02, OV-03, OV-04, OV-05
**Success Criteria** (what must be TRUE):
  1. ユーザーは120話の基本統計（総エピソード数、物語期間、ライフイベントカバレッジ、メインキャラクター数）を一目で確認できる
  2. ユーザーは主要登場人物5人の概要を閲覧できる
  3. ユーザーは3部構成（基礎編・葛藤編・統合編）の話数、期間、テーマを確認できる
  4. ユーザーはさくら回想シーンの全体分布（20回：基礎編9回、葛藤編6回、統合編5回）を確認できる
  5. ユーザーは12カテゴリのサブテーマ統計（転職、恋愛、家族等）を一覧できる

### Phase 2: 13章構造可視化
**Goal**: 13章構造の全体像と詳細タイムラインを閲覧可能にする
**Depends on**: Phase 1（全体統計のデータ構造を基盤として使用）
**Requirements**: CH-01, CH-02, CH-03, CH-04
**Success Criteria** (what must be TRUE):
  1. ユーザーは13章構造の全体像（各章の話数範囲、期間、テーマ）を一覧できる
  2. ユーザーは各章の詳細タイムラインを展開して閲覧できる
  3. ユーザーは章ごとの関連キャラクターとテーマを確認できる
  4. ユーザーは13章構造と3部構成のマッピング関係を理解できる（どの章がどの部に属するか）

### Phase 3: さくら回想シーン
**Goal**: さくらの回想シーン20回の詳細な分布と内容を可視化する
**Depends on**: Phase 1（全体統計で既に基本分布は表示）
**Requirements**: SA-01, SA-02, SA-03, SA-04
**Success Criteria** (what must be TRUE):
  1. ユーザーは20回の回想シーン全件をリストで閲覧できる（話数、テーマ、出典）
  2. ユーザーは回想シーンの3部構成別分布を視覚的なグラフ等で確認できる
  3. ユーザーは各回想シーンの詳細（テーマ、内容）をクリックして確認できる
  4. ユーザーは回想シーンを物語の流れで時系列順に追える

### Phase 4: サブテーマ統計
**Goal**: 12テーマ別の統計と3部間のテーマ推移を可視化する
**Depends on**: Phase 1（サブテーマ基本統計を基盤として使用）
**Requirements**: TH-01, TH-02, TH-03, TH-04
**Success Criteria** (what must be TRUE):
  1. ユーザーは12テーマ別の登場回数統計を閲覧できる（転職、恋愛、家族等）
  2. ユーザーは各テーマの割合と具体的な登場話を確認できる
  3. ユーザーは3部別（基礎編・葛藤編・統合編）のテーマ推移を視覚的に追える
  4. ユーザーは主要テーマの登場話例を参照できる
