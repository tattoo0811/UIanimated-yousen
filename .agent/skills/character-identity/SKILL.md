# 算命学キャラクター人格定義スキル

## 概要

「九条巡の運命診断室」の登場人物ひとりひとりに、算命学の命式に基づいた一貫性のある人格を定義する。

**2ファイル構成**:
- `identity.md` — **誰であるか**（命式・性格・話し方・外見）
- `soul.md` — **どう在るか**（天中殺・大運・成長弧・世界観）

> 参考: [IDENTITY.mdとSOUL.mdの設計パターン](https://zenn.dev/kei31ai/articles/20260210-ai-personality-design)

---

## 算命学ファースト原則

> **すべての性格・行動・世界観は算命学の命式から導出せよ。根拠なき設定は禁止。**

各性格特性には必ず以下のいずれかの「出典」を明記する:

| 出典 | 用途 |
|------|------|
| 日干（十干） | 人間の本質・元素イメージ |
| 十大主星（陽占5星） | 性格の5次元（中央/北/南/東/西） |
| 十二大従星（3星） | エネルギー配分・人生フェーズ |
| 五行バランス | 強み/弱み・行動傾向 |
| 天中殺 | 盲点・見えない課題 |
| 大運 | 人生の転換期・現在の運期 |
| 相生/相剋 | 内的葛藤・他者との関係性 |

---

## 巡の語りの原則 ⚠️

> **巡は患者に対して算命学用語を直接使わない。命式を「患者の人生の言葉」に翻訳して語る。**

### 口に出さないもの（算命学用語）

- 星の名前:「司禄星」「龍高星」「天将星」など
- 五行の数値:「木が128」「火がゼロ」
- 天中殺の名称:「子丑天中殺」「戌亥天中殺」
- 術語:「相剋」「木生火」「蔵干」

### 口に出すもの（患者の世界の言葉）

- 自然の比喩:「あなたの中には、ずっと育ちたかった力がありますね」
- 患者の人生に紐づけた例え:「SNSの数字を追いながら、『これじゃない』って思ってたでしょう？」
- 感覚の言語化:「苦しかったでしょう？」「花を咲かせたいのに、咲かせ方がわからなかったんじゃないですか」
- 季節や時間:「あなたの花は、これから咲くんですよ。今はまだ根を張る季節」

### identity.md / soul.md での書き分けルール

| セクション | 算命学用語 | 患者の言葉 |
|-----------|-----------|-----------|
| 命式・五行（資料セクション） | ✅ 使用OK | — |
| 元素の姿 / 内なる矛盾 | ✅ 使用OK（括弧内で出典明記） | — |
| 巡が見る風景 →「巡の内面」 | ✅ 地の文・心の声として使用OK | — |
| 巡が見る風景 →「語りかけ」 | ❌ 禁止 | ✅ 必須 |
| 話し方の例 | ❌ 禁止 | ✅ 必須 |
| 天中殺の活かし方（巡のアドバイス） | ❌ 禁止 | ✅ 必須 |

### 例: 変換パターン

| ❌ 算命学用語のまま | ✅ 患者の言葉に翻訳 |
|---|---|
| 「木が128で火がゼロです」 | 「育ちたい力はいっぱいあるのに、自分を表に出す力がまだ眠っているんです」 |
| 「あなたは天禄星×2です」 | 「地に足がついてる人ですね。安定してる。でもその安定が、少し窮屈でしょう？」 |
| 「子丑天中殺ですね」 | 「居場所がないって、ずっと感じてきたんじゃないですか？」 |
| 「司禄星が中央にある」 | 「あなたは大事なものをゆっくり溜め込んでいく人ですよ」 |

## 十大主星→性格特性変換ルール

各星が**どの位置**にあるかで意味が変わる:

| 位置 | 意味 |
|------|------|
| 中央 | 本質的な自分（日干と最も近い） |
| 北 | 目上・親への態度 |
| 南 | 目下・子供への態度 |
| 東 | 仲間・同僚への態度 |
| 西 | 配偶者・パートナーへの態度 |

| 星 | キーワード | 話し方への影響 |
|----|-----------|---------------|
| 貫索星 | 自立、頑固、マイペース | 「〜だと思う」が多い、断定調 |
| 石門星 | 協調、人間関係、交渉 | 「みんなで」「一緒に」が多い |
| 鳳閣星 | 楽観、食道楽、表現力 | のんびりした口調、ユーモア |
| 調舒星 | 繊細、芸術、孤独 | 比喩が多い、間を大事にする |
| 禄存星 | 奉仕、魅力、引力 | 相手を褒める、寄り添う |

| 車騎星 | 行動、闘争、単純 | 短文、命令調、「やるぞ」 |

## 使い方

### 1. 入力データの準備

`novel/characters/ep1-10.md`（または該当するエピソードファイル）から以下を抽出:
- 生年月日
- 命式（年柱/月柱/日柱）
- 日干
- 十大主星（5星）
- 十二大従星（3星）
- エネルギー値
- 天中殺
- 年表
- 物語における核

### 2. ファイル生成

テンプレートに従い、以下の2ファイルを生成:

```
novel/characters/personas/{character-name}/
├── identity.md   ← templates/identity-template.md に従う
└── soul.md        ← templates/soul-template.md に従う
```

**命名規則**: フォルダ名はローマ字（例: `takahashi-misaki`, `murata-kenichi`）

### 3. CLI必須ダブルチェック ⚠️

> **ソースデータ（ep1-10.md等）には既知の誤りがある。** 必ずCLIで検証すること。

```bash
bun run tools/rev-sanmei-cli-v6.ts {YYYY-MM-DD}
```

**確認項目**:
1. **命式**（年柱/月柱/日柱）がCLI出力と一致するか
2. **十大主星**（5星の位置と種類）がCLI出力と一致するか
3. **十二大従星**（3星と点数）がCLI出力と一致するか
4. **天中殺** — CLIの `calcTenchusatsu(日干, 日支)` から計算
5. **エネルギー値** がCLI出力と一致するか

**天中殺の判定方法**（CLIに表示されない場合）:
```
日干のGAN index と 日支のSHI index の差分 (sIdx - gIdx + 12) % 12:
  0 = 戌亥天中殺
  2 = 子丑天中殺
  4 = 寅卯天中殺
  6 = 辰巳天中殺
  8 = 午未天中殺
  10 = 申酉天中殺
```

**ダッシュボード確認**（利用可能な場合）:
```
http://localhost:3001/dashboard/characters/{character-id}
```

⚠️ CLI結果とソースデータに不一致がある場合、**CLI結果を正とし**、identity.md / soul.md に注記を付ける。

### 4. 品質チェック

生成後、以下を確認:

- [ ] すべての性格特性に算命学の出典が明記されているか
- [ ] 日干の元素メタファーが一貫しているか
- [ ] 話し方の例文が4つ以上あるか
- [ ] 内なる矛盾が相剋関係から説明されているか
- [ ] 天中殺の盲点が具体的に言語化されているか
- [ ] 成長弧が大運の転換と連動しているか
- [ ] **巡の語りの原則**: 語りかけ・話し方の例・天中殺アドバイスに算命学用語が含まれていないか ⚠️
- [ ] **巡が見る風景**: 「巡の内面」と「語りかけ」が分離されているか
